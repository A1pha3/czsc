# CZSC 项目核心工具

## 文档概述

本文档旨在帮助新用户快速了解 CZSC（缠中说禅技术分析工具）库中可用的实用工具，按模块分类列出所有核心功能，并提供使用场景和API参考。

### 如何使用本文档

- **系统学习**：按模块顺序阅读，了解CZSC的完整工具体系
- **按需查阅**：通过"按功能查找索引"快速定位所需工具
- **快速上手**：参考"快速上手指南"开始第一个项目

### 工具重要性标注

- **⭐ 必用高频** - 几乎每个CZSC项目都会用到，建议优先掌握
- **🔥 常用中频** - 大多数策略研究需要，重要程度次之
- **💡 特定场景** - 特定需求时使用，按需学习

## 1. 核心分析工具 (czsc/core.py)

### 模块概述

核心分析工具是 CZSC 库的基础，实现了缠论的分型、笔、线段等核心概念的自动识别，以及基于此的多级别量化交易策略。

### 工具列表

| 工具名称 | 重要性 | 功能说明 | 使用场景 |
|---------|--------|----------|----------|
| `CZSC` | ⭐ | 缠论分析核心类，实现分型、笔、线段识别 | 所有缠论分析的基础 |
| `RawBar` | ⭐ | 原始K线数据结构 | K线数据处理 |
| `NewBar` | ⭐ | 合成K线数据结构 | 多级别K线合成 |
| `Freq` | ⭐ | K线频率枚举（日线、30分钟等） | 指定K线周期 |
| `format_standard_kline` | ⭐ | DataFrame转RawBar列表 | 数据格式转换 |
| `BarGenerator` | ⭐ | K线生成器，支持多级别合成 | 多级别分析 |
| `Signal` | 🔥 | 信号数据结构 | 信号生成与传递 |
| `Event` | 🔥 | 事件数据结构 | 事件定义与匹配 |
| `Position` | 🔥 | 持仓数据结构 | 持仓管理 |
| `Operate` | 🔥 | 操作枚举（开多、平多等） | 交易操作定义 |
| `Direction` | 🔥 | 方向枚举（向上、向下） | 趋势方向判断 |
| `ZS` | 💡 | 中枢数据结构 | 中枢分析 |

### API 参考

```python
# CZSC 分析器
czsc_obj = CZSC(bars, max_bi_num=envs.get_max_bi_num())

# 数据格式转换
bars = format_standard_kline(df, freq=Freq.F30)

# K线生成器
bg = BarGenerator(base_freq=Freq.F30, freqs=[Freq.F30, Freq.FD, FreqFW])
bg.update(bars)
```

### 快速示例

```python
from czsc.core import CZSC, format_standard_kline, Freq
from czsc.mock import generate_symbol_kines

# 生成测试数据
df = generate_symbol_kines('000001', '30分钟', '20240101', '20240105')

# 转换为CZSC对象
bars = format_standard_kline(df, freq=Freq.F30)
czsc_obj = CZSC(bars)

# 访问分析结果
print(f"笔的数量: {len(czsc_obj.bi_list)}")
```

## 4. 缓存与性能 (czsc/utils/cache.py)

### 模块概述

提供高效的磁盘缓存机制，避免重复计算，显著提升性能。支持函数级装饰器和类级缓存管理。

### 工具列表

| 工具名称 | 重要性 | 功能说明 | 使用场景 |
|---------|--------|----------|----------|
| `disk_cache` | ⭐ | 磁盘缓存装饰器 | 函数结果缓存 |
| `DiskCache` | ⭐ | 缓存管理类 | 高级缓存控制 |
| `home_path` | 🔥 | 缓存目录路径 | 缓存位置管理 |
| `clear_cache` | 🔥 | 清空所有缓存 | 定期清理 |
| `clear_expired_cache` | 🔥 | 清理过期缓存 | 自动维护 |
| `get_dir_size` | 💡 | 获取目录大小 | 缓存监控 |
| `empty_cache_path` | 💡 | 清空缓存目录 | 一键清理 |

### API 参考

```python
# 使用装饰器缓存函数结果
@disk_cache(cache_key="my_func_{arg1}_{arg2}")
def my_expensive_function(arg1, arg2):
    # 耗时计算
    return result

# 缓存管理类
cache = DiskCache()
cache.set("key", value, expire=3600)
result = cache.get("key")

# 清理缓存
clear_cache()  # 清空所有
clear_expired_cache()  # 清理过期
```

### 快速示例

```python
from czsc.utils import disk_cache

@disk_cache(cache_key="calc_{symbol}_{days}", expire=86400)
def calculate_indicators(symbol, days):
    # 耗时计算
    return result

# 第一次调用会计算并缓存
result1 = calculate_indicators("000001", 30)
# 第二次调用直接从缓存读取
result2 = calculate_indicators("000001", 30)
```

    daily_performance,
    top_drawdowns,
    rolling_daily_performance,

    psi,

### 快速示例

```python
from czsc.utils.analysis import (
    daily_performance,
    top_drawdowns,
    cross_sectional_ic
)

# 1. 计算策略绩效指标
returns = [0.01, 0.02, -0.01, 0.03, 0.02, -0.02, 0.01]
stats = daily_performance(returns)

print("=== 策略绩效 ===")
print(f"年化收益: {stats['年化']:.2%}")
print(f"夏普比率: {stats['夏普']:.2f}")
print(f"最大回撤: {stats['最大回撤']:.2%}")
print(f"卡玛比率: {stats['卡玛']:.2f}")
print(f"日胜率: {stats['日胜率']:.2%}")

# 2. 分析最大回撤
drawdowns = top_drawdowns(returns, top_n=3)
print("\n=== 最大回撤 ===")
for i, dd in enumerate(drawdowns, 1):
    print(f"第{i}大回撤: {dd['depth']:.2%}")

# 3. 计算因子IC
import pandas as pd
factor_df = pd.DataFrame(...)  # 因子数据
return_df = pd.DataFrame(...)   # 收益数据
ic = cross_sectional_ic(factor_df, return_df)
print(f"\n=== 因子IC ===")
print(f"IC均值: {ic.mean():.4f}")
print(f"IC标准差: {ic.std():.4f}")
print(f"ICIR: {ic.mean()/ic.std():.4f}")
```

## 快速上手指南

### 典型工作流程

一个完整的 CZSC 工作流程通常包含以下步骤：

```python
# 1. 数据获取
from czsc.utils import DataClient
dc = DataClient(symbol="000001", sdt="20240101", edt="20240105")
df = dc.get_kline()

# 2. 数据处理
from czsc.core import format_standard_kline, CZSC, Freq
bars = format_standard_kline(df, freq=Freq.F30)

# 3. 缠论分析
czsc_obj = CZSC(bars)

# 4. 可视化
from czsc.utils.plotly_plot import plot_czsc_chart
plot_czsc_chart(czsc_obj, show_bi=True, show_xd=True)
```

### 工具使用顺序

**必需顺序**
```
数据获取 (DataClient) → 数据转换 (format_standard_kline) → 分析 (CZSC) → 可视化/回测
```

**独立使用**
- 缓存工具 (`disk_cache`) 可在任何耗时函数上使用
- 技术指标 (`ta` 模块) 可独立计算
- 统计工具 (`eda` 模块) 可单独使用

## 常见使用场景

### 场景1：单品种策略回测

**涉及工具**：`DataClient` → `CZSC` → `CzscTrader` → `WeightBacktest`

**适用场景**：验证单个品种的交易策略

```python
from czsc.utils import DataClient
from czsc.core import CZSC, format_standard_kline, Freq
from czsc.traders import CzscTrader

# 获取数据并分析
dc = DataClient(symbol="000001", sdt="20230101", edt="20231231")
df = dc.get_kline()
bars = format_standard_kline(df, freq=Freq.F30)
czsc_obj = CZSC(bars)

# 创建交易员并回测
trader = CzscTrader(signal_config, pos_config)
results = trader.backtest(czsc_obj)
```

### 场景2：多品种组合优化

**涉及工具**：`CTAResearch` → `OpensOptimize` / `ExitsOptimize`

**适用场景**：多品种参数优化

```python
from czsc.sensors import CTAResearch

# 创建研究框架
cta = CTAResearch(symbols=["000001", "000002"], sdt="20230101", edt="20231231")

# 参数优化
param_grid = {'open_para': [1, 2, 3], 'exit_para': [0.5, 1.0]}
results = cta.optimize(param_grid)
```

### 场景3：因子研究与分析

**涉及工具**：`cross_sectional_ic` → `judge_factor_direction` → `remove_beta_effects`

**适用场景**：评估因子的有效性

```python
from czsc.utils import cross_sectional_ic
from czsc.eda import judge_factor_direction, remove_beta_effects

# 计算IC
ic = cross_sectional_ic(factor_df, return_df)

# 判断方向
direction = judge_factor_direction(factor_df, return_df)

# 因子中性化
neutral_factor = remove_beta_effects(factor_df, return_df)
```

### 场景4：实时监控与可视化

**涉及工具**：`DataClient` → `BarGenerator` → `plot_czsc_chart`

**适用场景**：实时数据可视化

```python
from czsc.utils import DataClient
from czsc.utils.plotly_plot import plot_czsc_chart

# 获取实时数据
dc = DataClient(symbol="000001")
df = dc.get_kline()

# 可视化
bars = format_standard_kline(df, freq=Freq.F30)
czsc_obj = CZSC(bars)
plot_czsc_chart(czsc_obj, show_bi=True, show_xd=True)
```

## 版本说明与兼容性

### Rust/Python 混合架构
- **版本控制**：通过环境变量 `CZSC_USE_PYTHON` 控制
- **默认行为**：优先使用 Rust 版本（rs-czsc），性能更优
- **回退机制**：Rust 版本不可用时自动回退到 Python 版本

### 环境变量
```bash
# 强制使用 Python 版本
export CZSC_USE_PYTHON=1

# 缠论参数
export czsc_min_bi_len=6  # 最小笔长度
export czsc_max_bi_num=50  # 最大笔数量
```

- Python 3.8+
- 推荐使用 UV 进行依赖管理
