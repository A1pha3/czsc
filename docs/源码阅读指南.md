## 项目简介

**CZSC** 是基于缠中说禅理论的综合性量化交易 Python 库，提供：

- ✅ 缠论核心概念（分型、笔、线段、中枢）的自动识别
- ✅ 多级别联立分析框架
- ✅ 信号-事件-交易的完整量化体系
- ✅ 策略回测和参数优化工具
- ✅ 多数据源接入（Tushare、聚宽、天勤等）
- ✅ Rust/Python 混合架构（性能优化）

**技术栈**: Python 3.8+, Rust (可选), Pandas, Plotly, Streamlit

## 核心概念理解

### 信号-事件-交易体系

这是 CZSC 最核心的量化交易方法论：

```
原始K线 → 信号(Signal) → 事件(Event) → 交易决策(Position)
```

#### 1. **信号 (Signal)**
- 最基础的技术指标或市场状态判断
- 例如：均线位置、分型状态、成交量特征
- 位置：`czsc/signals/`
- 信号是最小的判断单元，每个信号函数返回一个明确的市场状态

#### 2. **事件 (Event)**
- 通过逻辑关系（AND/OR/NOT）组合多个信号
- 代表一个完整的市场事件或交易条件
- 例如："多头启动事件" = "MA5上穿MA20"(signals_all) AND NOT "涨停"(signals_not)
- 对应开仓或平仓的触发条件
- 包含三种信号组合方式：
  - `signals_all`: 必须全部满足的信号列表
  - `signals_any`: 满足任一即可的信号列表  
  - `signals_not`: 不能出现的信号列表

#### 3. **交易 (Position)**
- 基于事件的完整交易策略
- 定义开仓事件列表（opens）和平仓事件列表（exits）
- 包含止损、超时、开仓间隔等风险控制参数

### 缠论核心对象

```python
RawBar   # 原始K线（带 OHLCV）
  ↓ 去除包含关系
NewBar   # 处理后的K线（无包含关系）
  ↓ 识别分型
FX       # 分型（顶分型/底分型）
  ↓ 识别笔
BI       # 笔（连接相邻分型的线段）
  ↓ 识别线段和中枢
ZS       # 中枢（价格震荡区间）
```

**说明**: 这些是缠论的核心技术分析对象，与交易决策体系（Signal→Event→Position）是两个不同的层面。

### 多级别联立分析

CZSC 支持同时分析多个时间周期，例如：

```python
freqs = ['1分钟', '5分钟', '30分钟', '日线']
# 可以组合不同级别的信号进行决策
# 例如：日线看趋势 + 30分钟看入场点 + 1分钟看精确位置
```

## 详细模块解析

### 模块1: `czsc/py/analyze.py` - 缠论分析核心

**核心类**: `CZSC`

**关键方法**:

```python
class CZSC:
    def __init__(self, bars: List[RawBar], **kwargs):
        """初始化CZSC分析对象"""
        
    def update(self, bar: RawBar):
        """更新一根K线，实时识别分型、笔"""
        
    @property
    def bars_raw(self) -> List[RawBar]:
        """原始K线序列"""
        
    @property
    def bars_ubi(self) -> List[NewBar]:
        """去除包含关系后的K线序列"""
        
    @property
    def fx_list(self) -> List[FX]:
        """识别出的分型序列"""
        
    @property
    def bi_list(self) -> List[BI]:
        """识别出的笔序列"""
```

**工作流程**:
1. 输入原始K线 (`RawBar`)
2. 去除包含关系 → 生成 `NewBar` 序列
3. 识别分型 → 生成 `FX` 序列
4. 识别笔 → 生成 `BI` 序列
5. 识别中枢 → 生成 `ZS` 序列（如需要）

### 模块2: `czsc/traders/base.py` - 交易执行框架

**核心类**: `CzscSignals` 和 `CzscTrader`

#### `CzscSignals`: 多级别信号计算

```python
class CzscSignals:
    def __init__(self, bg: BarGenerator, **kwargs):
        """
        bg: K线生成器（包含多个周期的K线）
        signals_config: 信号配置列表
        """
        self.kas = {freq: CZSC(bars) for freq, bars in bg.bars.items()}
        # kas 是一个字典，key是周期，value是CZSC对象
        
    def get_signals_by_conf(self) -> OrderedDict:
        """根据配置计算所有信号"""
```

**使用示例**:
```python
signals_config = [
    {'name': 'czsc.signals.tas_ma_base_V221101', 
     'freq': '日线', 'di': 1, 'ma_type': 'SMA', 'timeperiod': 5},
    {'name': 'czsc.signals.cxt_bi_status_V230101', 
     'freq': '30分钟', 'di': 1},
]

cs = CzscSignals(bg, signals_config=signals_config)
signals = cs.s  # 当前所有信号字典
```

#### `CzscTrader`: 完整交易决策

```python
class CzscTrader:
    def __init__(self, bg: BarGenerator, positions: List[Position], **kwargs):
        """
        bg: K线生成器
        positions: 持仓策略列表
        """
        
    def update(self, bar: RawBar):
        """更新一根K线，判断是否触发交易事件"""
        
    @property
    def positions(self) -> List[Position]:
        """所有持仓策略及其状态"""
```

### 模块3: `czsc/strategies.py` - 策略定义

**策略四要素**:

1. **交易品种**: `symbol` 和品种相关参数
2. **K线周期**: 从信号配置中自动提取
3. **信号配置**: `signals_config` 列表
4. **持仓策略**: `positions` 方法返回的 `Position` 列表

**标准流程**:
```python
class MyStrategy(CzscStrategyBase):

                        "signals_not": ["信号5"],           # 不能出现
                    }
                ]
            }
        ]
        
        # 定义平仓事件
        exits = [...]
        
        # 创建持仓对象
        pos = Position(name="策略名", symbol=self.symbol, 
                      opens=[Event.load(x) for x in opens],
                      exits=[Event.load(x) for x in exits])
        return [pos]
```

## 常见问题 FAQ

### Q1: 如何选择阅读Python版本还是Rust版本？

**A**: 
- **初学者**: 建议阅读 **Python 版本**（`czsc/py/`），代码更易理解
- 设置环境变量 `CZSC_USE_PYTHON=1` 强制使用 Python 版本
- Rust 版本主要用于性能优化，核心逻辑与 Python 版本一致

### Q2: 信号函数中的 `di` 参数是什么意思？

**A**: `di` 表示"倒数第几根K线"（Distance Index）
- `di=1`: 最后一根K线（实时）
- `di=2`: 倒数第二根K线
- 用于回溯分析历史信号

### Q3: 如何理解"去除包含关系"？

**A**: 这是缠论的核心概念
- 包含关系: 一根K线的高低点被另一根完全包含
- 去除方法: 根据趋势方向合并K线（向上取大，向下取小）
- 详见 `czsc/py/analyze.py` 中的 `remove_include` 函数

### Q4: 策略回测为什么没有交易信号？

**A**: 检查以下几点：
1. K线数据是否充足（至少100根以上）
2. 信号配置是否正确（检查信号名称和参数）
3. 开仓事件的信号条件是否过于严格
4. 使用 `print(cs.s)` 查看实际生成的信号

### Q5: 如何获取更多数据源？

**A**: 
- 查看 `czsc/connectors/` 目录
- Tushare: 需要 token（免费注册）
- 聚宽: 需要账号认证
- 天勤: 期货数据，需要账号
- 建议先使用 `czsc.mock.generate_symbol_kines` 模拟数据学习
