## 安装

### 方式一：使用 UV（推荐）

UV 是一个快速的 Python 包管理器。

```bash
# 安装 UV（如果还没安装）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 克隆项目
git clone https://github.com/waditu/czsc.git
cd czsc

# 安装依赖
uv sync --extra dev
```

### 方式二：使用 pip

```bash
# 从 PyPI 安装
pip install czsc -U

# 或从 GitHub 安装最新版
pip install git+https://github.com/waditu/czsc.git -U
```

### 验证安装

```bash
python -c "import czsc; print(f'CZSC version: {czsc.__version__}')"
```

### 常见安装问题

#### 问题 1：`rs-czsc` 安装失败

```
error: can't find Rust compiler
```

**原因**：rs-czsc 是 Rust 编写的性能优化模块，需要 Rust 编译环境。

**解决方案**：
```bash
# 方案 A：安装 Rust 编译器
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
pip install czsc -U  # 重新安装

# 方案 B：跳过 Rust，使用纯 Python 版本
export CZSC_USE_PYTHON=1
pip install czsc -U
```

#### 问题 2：Python 版本不兼容

```
ERROR: czsc requires Python >=3.10
```

**解决方案**：升级 Python 到 3.10+ 或使用 UV 管理多版本环境。

如果显示版本号，说明安装成功！

## 第一个示例

### 示例 1：识别分型和笔

```python
from czsc.core import CZSC, format_standard_kline, Freq
from czsc.mock import generate_symbol_kines

# 生成测试数据（30分钟K线）
df = generate_symbol_kines('000001', '30分钟', '20240101', '20240131')

# 转换为CZSC对象需要的格式
bars = format_standard_kline(df, freq=Freq.F30)

# 创建CZSC分析对象
czsc_obj = CZSC(bars)

# 查看分析结果
print(f"原始K线数量: {len(czsc_obj.bars_raw)}")
print(f"分型数量: {len(czsc_obj.fx_list)}")
print(f"笔数量: {len(czsc_obj.bi_list)}")

# 查看最后一笔
if czsc_obj.bi_list:
    last_bi = czsc_obj.bi_list[-1]
    print(f"\n最后一笔:")
    print(f"  方向: {last_bi.direction}")
    print(f"  起点: {last_bi.fx_a.dt}, 价格: {last_bi.fx_a.high}")
    print(f"  终点: {last_bi.fx_b.dt}, 价格: {last_bi.fx_b.low}")
```

**运行结果**：
```
原始K线数量: 640
分型数量: 124
笔数量: 41

最后一笔:
  方向: Direction.Down
  起点: 2024-01-30 10:00:00, 价格: 12.56
  终点: 2024-01-31 13:30:00, 价格: 12.38
```

### 示例 2：多级别K线合成

```python
from czsc.core import BarGenerator, format_standard_kline, Freq
from czsc.mock import generate_symbol_kines

# 准备1分钟K线数据
df = generate_symbol_kines('000001', '1分钟', '20240101', '20240105')
bars = format_standard_kline(df, freq=Freq.F1)

# 创建K线生成器，合成多个周期
bg = BarGenerator(
    base_freq='1分钟',
    freqs=['5分钟', '15分钟', '30分钟', '日线']
)

# 逐根更新K线
for bar in bars:
    bg.update(bar)

# 查看各周期K线数量
for freq, freq_bars in bg.bars.items():
    print(f"{freq}: {len(freq_bars)} 根K线")
```

### 示例 3：计算交易信号

```python
from czsc.traders import CzscSignals
from czsc.core import BarGenerator, format_standard_kline, Freq
from czsc.mock import generate_symbol_kines

# 准备数据
df = generate_symbol_kines('000001', '1分钟', '20240101', '20240105')
bars = format_standard_kline(df, freq=Freq.F1)

# 创建K线生成器
bg = BarGenerator(base_freq='1分钟', freqs=['5分钟', '30分钟', '日线'])
for bar in bars:
    bg.update(bar)

# 定义信号配置
signals_config = [
    # 均线信号
    {'name': 'czsc.signals.tas_ma_base_V221101',
     'freq': '日线', 'di': 1, 'ma_type': 'SMA', 'timeperiod': 5},
    # 分型状态信号
    {'name': 'czsc.signals.cxt_fx_power_V221107',
     'freq': '30分钟', 'di': 1},
]

# 创建信号计算对象
cs = CzscSignals(bg, signals_config=signals_config)

# 查看当前信号
print("当前信号:")
for k, v in cs.s.items():
    print(f"  {k}: {v}")
```

## 核心概念

### 信号-事件-交易体系

CZSC 使用三层量化交易体系：

```
原始K线 → 信号(Signal) → 事件(Event) → 交易决策(Position)
```

**信号 (Signal)**：最基础的技术指标判断
- 例如：均线方向、分型状态、成交量特征

**事件 (Event)**：信号的逻辑组合
- `signals_all`：必须全部满足
- `signals_any`：满足任一即可
- `signals_not`：不能出现

**交易 (Position)**：完整的交易策略
- 包含开仓条件、平仓条件、止损、超时等

### 缠论核心对象

```
RawBar   # 原始K线
  ↓ 去除包含关系
NewBar   # 处理后的K线
  ↓ 识别分型
FX       # 分型（顶分型/底分型）
  ↓ 识别笔
BI       # 笔（连接相邻分型）
  ↓ 识别中枢
ZS       # 中枢（价格震荡区间）
```

## 常见问题

### Q1: 如何获取真实数据？

A: 使用 DataClient 获取数据（基于 Tushare 数据源）：

```python
from czsc.utils.data import DataClient

# DataClient 是 Tushare 风格的数据客户端
# 需要先在 https://tushare.pro 注册获取 token
dc = DataClient(token="your_tushare_token")

# 获取日线数据
df = dc.post_request(
    api_name='daily',
    ts_code='000001.SZ',
    start_date='20240101',
    end_date='20240131'
)
```

支持的数据源：Tushare、聚宽、天勤、CCXT 等。详见[数据获取与处理指南](数据获取与处理指南.md)。

### Q2: 如何编写自定义信号？

A: 参考信号函数模板：

```python
from collections import OrderedDict
from typing import Union

def my_signal_V240101(c: CZSC, **kwargs) -> OrderedDict:
    """自定义信号函数

    参数模板: "30分钟_信号名称_参数1_参数2_0"

    :param c: CZSC对象
    :param kwargs: 其他参数
    :return: 信号字典
    """
    di = kwargs.get('di', 1)

    # 信号判断逻辑
    v = "满足" if 条件 else "不满足"

    s = {
        "信号名称": f"{c.freq.value}_D{di}_信号名称_参数1_参数2_{v}_0"
    }
    return s
```

### Q3: 如何进行回测？

A: 使用 CTAResearch 框架：

```python
from czsc.sensors import CTAResearch

cta = CTAResearch(
    strategy=MyStrategy,
    read_bars=read_bars_function,
    results_path='./results'
)

cta.replay(symbol='000001', sdt='20230101', edt='20231231')
```
