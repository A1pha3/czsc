## 一、策略开发概述

### 1.1 什么是 CZSC 策略？

**CZSC 策略**是基于缠论信号的完整交易系统，包含以下要素：

```
┌─────────────────────────────────────────────────────────┐
│                     CZSC 策略结构                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 品种参数        → 交易标的、基础周期                │
│  2. K线周期         → 多周期配置                        │
│  3. 信号配置         → 使用的信号函数                   │
│  4. 持仓策略         → 开仓、持仓、平仓逻辑             │
│  5. 风控参数         → 止损、止盈、仓位管理             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 策略开发的三大原则

| 原则 | 说明 | 实践 |
|------|------|------|
| **客观性** | 策略逻辑明确，可量化 | 使用明确的信号定义 |
| **可回测** | 历史数据可验证 | 编写回测代码 |
| **可执行** | 能够实盘运行 | 考虑滑点、手续费 |

### 1.3 策略开发流程

```
1. 需求分析
   ↓
2. 信号设计
   ↓
3. 策略编码
   ↓
4. 回测验证
   ↓
5. 参数优化
   ↓
6. 实盘测试
   ↓
7. 上线运行
```

## 二、CzscStrategyBase 策略基类

### 2.1 基类结构

```python
from abc import ABC, abstractmethod
from typing import List
from czsc.core import RawBar, Position

class CzscStrategyBase(ABC):
    """择时交易策略基类

    策略要素：
    1. 交易品种参数
    2. K线周期列表
    3. 信号配置
    4. 持仓策略列表
    """

    def __init__(self, **kwargs):
        self.kwargs = kwargs
        self.signals_module_name = kwargs.get("signals_module_name", "czsc.signals")

    @property
    def symbol(self) -> str:
        """交易标的"""
        return self.kwargs["symbol"]

    @property
    def freqs(self) -> List[str]:
        """K线周期列表"""
        # 从信号配置中提取
        pass

    @property
    def base_freq(self) -> str:
        """基础K线周期"""
        # 最小周期
        pass

    @abstractmethod
    def positions(self) -> List[Position]:
        """持仓策略列表（必须实现）"""
        raise NotImplementedError
```

### 2.2 策略属性说明

```python
"""
```

### 3.2 Event 事件定义

```python
from czsc.core import Event, Signal, Operate

event = Event(
    name="三买开仓事件",
    operate=Operate.LO,           # 操作类型：LO=开多, SO=开空

    # 必须全部满足的信号（AND 逻辑）
    # Signal 构造格式：signal="k1_k2_k3_v1_v2_v3_score"
    # 其中“任意”可作为通配符匹配任何值
    signals_all=[
        Signal(signal="日线_cxt_third_buy_符合_任意_任意_0"),
        Signal(signal="日线_tas_macd_base_红柱_任意_任意_0"),
    ],

    # 满足任一即可的信号（OR 逻辑）
    signals_any=[
        Signal(signal="30分钟_cxt_first_buy_符合_任意_任意_0"),
        Signal(signal="30分钟_cxt_second_buy_符合_任意_任意_0"),
    ],

    # 不能出现的信号（NOT 逻辑）
    signals_not=[
        Signal(signal="日线_bar_zdt_涨停_任意_任意_0"),
    ],
)
```

### 3.3 事件逻辑说明

```python
"""
事件执行逻辑：

1. signals_all：所有信号都必须满足
   if all(signal in current_signals for signal in event.signals_all):
       # 条件1通过

2. signals_any：至少一个信号满足
   if any(signal in current_signals for signal in event.signals_any):
       # 条件2通过

3. signals_not：所有信号都不能出现
   if not any(signal in current_signals for signal in event.signals_not):
       # 条件3通过

4. 最终判断：
   if (signals_all满足) and (signals_any满足) and (signals_not不满足):
       # 触发事件
"""
```

### 4.1 实例一：简单的三买策略

```python
from czsc.strategies import CzscStrategyBase
from czsc.core import Position, Event, Operate
from czsc.py.enum import Direction

class ThirdBuyStrategy(CzscStrategyBase):
    """缠论第三类买点策略

    策略逻辑：
    1. 日线级别出现三买
    2. MACD 红柱
    3. 30分钟级别一买或二买共振
    """

    @property
    def positions(self) -> List[Position]:
        """定义持仓策略"""
        # 开仓事件
        open_event = Event(
            name="三买开多",
            operate=Operate.LO,
            signals_all=[
                "日线_cxt_third_buy_符合_0",
                "日线_tas_macd_base_红柱_0",
            ],
            signals_any=[
                "30分钟_cxt_first_buy_符合_0",
                "30分钟_cxt_second_buy_符合_0",
            ],
            signals_not=[
                "日线_bar_zdt_涨停_0",
            ]
        )

        # 止损事件：跌破前低
        stop_event = Event(
            name="止损",
            operate=Operate.SO,
            signals_all=[
                "30分钟_cxt_bi_stop_符合_0",
            ],
            signals_any=[],
            signals_not=[]
        )

        # 平仓事件：三卖
        exit_event = Event(
            name="平仓",
            operate=Operate.SO,
            signals_all=[
                "日线_cxt_third_sell_符合_0",
            ],
            signals_any=[],
            signals_not=[]
        )

        position = Position(
            name="三买策略",
            symbol=self.symbol,
            opens=[open_event],
            stops=[stop_event],
            exits=[exit_event],
            holds=[],
            timeout=30,
        )

        return [position]
```

### 6.2 信号权重优化

```python
def optimize_signal_weights(signals_list, bars):
    """优化信号权重

    参数：
        signals_list: 候选信号列表
        bars: K线数据

    返回：
        最佳信号组合
    """
    # 计算每个信号的预测能力
    from czsc.utils import cross_sectional_ic

    signal_scores = {}
    for signal in signals_list:
        # 计算信号的IC值
        ic = cross_sectional_ic(signal, bars)
        signal_scores[signal] = abs(ic)

    # 选择IC值最高的信号
    sorted_signals = sorted(signal_scores.items(), key=lambda x: x[1], reverse=True)

    return sorted_signals[:5]  # 返回前5个信号
```

## 八、最佳实践

### 8.1 策略设计原则

```python
"""
1. 简单性原则
   - 避免过度复杂的逻辑
   - 每个信号都有明确含义

2. 鲁棒性原则
   - 在不同市场环境下都能工作
   - 对参数变化不敏感

3. 可解释性原则
   - 每个交易决策都有明确依据
   - 能够解释亏损原因

4. 可执行性原则
   - 考虑滑点和手续费
   - 考虑流动性限制
"""
```

### 8.2 常见错误避免

```python
"""
常见错误：

1. 过度拟合
   - 使用过多参数
   - 针对历史数据优化

2. 数据窥探
   - 使用未来信息
   - 频繁修改策略

3. 忽略成本
   - 未考虑手续费
   - 未考虑滑点

4. 风险控制不足
   - 没有止损
   - 仓位过重
"""
```

```python
            stops=[stop_event],
            exits=[exit_event],
            holds=[],
            timeout=self.param1,
        )

        return [position]
```
