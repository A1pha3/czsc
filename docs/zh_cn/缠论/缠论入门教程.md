## 一、为什么要学缠论？

### 1.1 一个真实的困惑

假设你是一位股票投资者，你每天面对下面这些问题：

- 均线金叉了，但上次金叉我亏了 30%，这次能信吗？
- 大 V 说要涨，朋友说要跌，该听谁的？
- 我知道"趋势是你的朋友"，但到底什么时候趋势开始、什么时候结束？

这些问题的根源在于：**传统技术分析的结论依赖主观判断**。同一张 K 线图，十个人可能画出十种趋势线。缠论尝试解决的正是这个问题——它提供了一套**客观的、可计算的**市场分析框架。

### 1.2 缠论 vs 传统技术分析

| 维度 | 传统技术分析 | 缠论 |
|------|-------------|------|
| **判断标准** | 主观（"看起来像头肩顶"） | 客观（严格的数学定义） |
| **买卖点** | 模糊（"支撑位附近"） | 精确（分型级别的定位） |
| **多周期** | 人工对比多张图 | 数学化的级别递归 |
| **可回测性** | 弱（人工标注不一致） | 强（算法输出唯一确定） |
| **学习曲线** | 低（但天花板也低） | 高（但上限极高） |

> 💡 **类比理解**：传统技术分析像"看云识天气"——经验丰富的人有时很准，但无法标准化传授。缠论更像"气象学"——用温度、气压、湿度等客观指标来预测天气，虽然不能保证 100% 准确，但方法可复现、可改进。

### 1.3 缠论的局限性

在深入学习之前，先建立正确的预期——这会帮你避免走弯路：

| 缠论能做的 | 缠论不能做的 |
|------------|-------------|
| 客观描述当前走势的结构 | 预测明天涨还是跌 |
| 提供高概率的买卖点 | 保证每次交易都盈利 |
| 多级别综合分析 | 替代风险管理和仓位控制 |
| 程序自动识别（可回测） | 在数据不足时给出可靠结论 |

> 💡 **建立正确心态**：缠论是一把精密的"手术刀"——它能精确地分析走势结构，但能否赚钱取决于使用者的整体交易系统（包括仓位管理、风控、心理控制等）。就像一把好刀不能保证每道菜都好吃。

## 二、缠论的核心哲学

### 2.1 第一公理：走势终完美

**"走势终完美"**是缠论的第一公理，也是整个理论大厦的基石。

> **一句话理解**：任何级别的上涨或下跌，都不会永远持续下去，最终一定会结束。

这似乎是一句废话？不，它有三层递进的深刻含义：

```
推理链条：

  走势终完美（公理）
       │
       ├──① 趋势有始有终
       │     一段上涨必然有起点和终点
       │     → 那我们能不能"在接近终点时行动"？
       │          │
       │          ├──② 结构可分解
       │          │     走势的内部分为更小的片段
       │          │     → 如果能识别这些片段的"完成"信号呢？
       │          │          │
       │          │          └──③ 买卖点可推导
       │          │                通过分析结构的完成度
       │          │                找到高概率的转折点
       │          │
       │          └── 这就是分型→笔→中枢的分析目标！
       │
       └── 从一个"废话"推导出了整套分析方法
```

**每一层的实际意义**：

1. **趋势有始有终**：无论多强的牛市，最终会见顶回落。这不是预测，而是事实。缠论的任务是尽早识别这个转折。
2. **结构可分解**：一段日线级别的上涨，内部一定有 30 分钟级别的"上→下→上→下……"波动。分析这些波动的特征，就能推断整段上涨的"健康状况"。
3. **买卖点可推导**：当我们能量化走势的"完成度"时，就能客观地定义买卖点——这正是后面要学的三类买卖点。

#### 生活类比

想象一座山：

```
        ∧
       / \        这座"山"有多高不确定
      /   \       但它一定有顶（走势终完美）
     /     \      山的两侧有坡度变化（结构可分解）
    /       \     最高点就是我们要找的"卖点"
___/         \___
```

无论这座山有多高、多陡，它一定有顶峰。缠论的任务就是在"爬山"过程中，用客观标准判断"是否已经接近山顶"。

### 2.2 第二公理：级别递归

**"级别递归"**是缠论的第二个核心思想：大级别的走势由小级别的走势递归构成。

> **一句话理解**：日线的一笔，内部是由 30 分钟级别的完整走势构成的。

#### 生活类比——俄罗斯套娃

```
┌──── 月线走势 ──────────────────────────┐
│  ┌── 周线走势 ────────────────────┐    │
│  │  ┌── 日线走势 ──────────┐      │    │
│  │  │  ┌── 30分钟走势 ─┐   │      │    │
│  │  │  │  5分钟走势...  │   │      │    │
│  │  │  └───────────────┘   │      │    │
│  │  └──────────────────────┘      │    │
│  └────────────────────────────────┘    │
└────────────────────────────────────────┘
```

就像你可以用乐高积木搭建一栋大楼——每块小积木（5 分钟走势）组成墙壁（30 分钟走势），墙壁组成楼层（日线走势），楼层组成大楼（周线走势）。

这个思想带来的实践意义是：
- **大级别看方向**：周线向上，就顺着做多
- **中级别找机会**：日线出现买点
- **小级别精确入场**：30 分钟级别确认后入场

### 2.3 第三公理：分类是客观的

缠论对走势进行了**穷举式分类**：在任何时刻，走势只能处于以下三种状态之一：

| 状态 | 定义 | 操作建议 |
|------|------|----------|
| **上涨趋势** | 至少两个依次升高的中枢 | 持有/做多 |
| **下跌趋势** | 至少两个依次降低的中枢 | 回避/做空 |
| **盘整** | 只有一个中枢 | 观望/高抛低吸 |

> **关键洞察**：不存在第四种可能。这意味着我们不需要"预测"市场走向，只需要**分类**当前走势属于哪种状态，然后采取对应的操作策略。

## 三、缠论的五个核心概念

缠论的分析过程是**自底向上**的层级构建，就像搭积木一样：

```
Level 5   中枢（ZS）     ← 由多笔重叠构成
Level 4   笔（BI）       ← 连接相邻分型
Level 3   分型（FX）     ← 由3根无包含K线组成
Level 2   无包含K线      ← 去除包含关系后的K线
Level 1   原始K线        ← 市场原始数据
```

> 💡 **学习路线**：务必从下往上学习，每个概念都建立在前一个概念之上。跳过任何一层都会导致后续的理解困难。

### 3.1 原始 K 线（Level 1）

原始 K 线是市场交易数据的最基本表示。每根 K 线记录了一个时间周期内的四个价格和两个量：

```
    │ ← 最高价（High）
    │
┌───┤ ← 开盘价（Open）或收盘价（Close）
│   │
│ 实 │   收阳线：Close > Open（通常显示红色）
│ 体 │   收阴线：Close < Open（通常显示绿色）
│   │
└───┤ ← 收盘价（Close）或开盘价（Open）
    │
    │ ← 最低价（Low）

    附加信息：成交量（Vol）、成交额（Amount）
```

在 CZSC 中，原始 K 线用 `RawBar` 对象表示：

```python
from czsc.core import RawBar, Freq
from datetime import datetime

# 一根30分钟K线
bar = RawBar(
    symbol="000001",         # 标的代码（如平安银行）
    id=1,                    # K线编号（必须升序）
    dt=datetime(2024, 1, 2, 10, 0),  # 时间
    freq=Freq.F30,           # 频率：30分钟
    open=10.0,               # 开盘价
    close=10.5,              # 收盘价
    high=10.8,               # 最高价
    low=9.9,                 # 最低价
    vol=10000,               # 成交量
    amount=105000,           # 成交额
)
```

### 3.2 包含关系与无包含 K 线（Level 2）

#### 什么是包含关系？

当两根 K 线的高低点存在"包含"时，我们说它们具有**包含关系**：

```
包含关系的两种情况：

情况1: K2 包含 K1          情况2: K1 包含 K2
   K2.High                    K1.High
    │                          │
  ┌─┤                        ┌─┤
  │ │  K1.High               │ │  K2.High
  │ ├──┤                     │ ├──┤
  │ │  │                     │ │  │
  │ │  │                     │ │  │
  │ ├──┤                     │ ├──┤
  │ │  K1.Low                │ │  K2.Low
  └─┤                        └─┤
    │                          │
   K2.Low                     K1.Low

判断条件（任一成立即为包含）：
  K2.High >= K1.High 且 K2.Low <= K1.Low  （K2包含K1）
  K1.High >= K2.High 且 K1.Low <= K2.Low  （K1包含K2）
```

#### 为什么要处理包含关系？

> **生活类比**：想象你在看一个人走路的轨迹。如果他走了一步前进 3 米，又走一步前进了 2 米但同时也后退了 1 米（=包含关系），你就搞不清他到底是在"前进"还是"犹豫"。去除包含关系，就是把这种模糊步伐"标准化"——确定他每一步的明确方向。

包含关系的 K 线会干扰分型的判断。考虑以下例子：

```
没有包含关系时，方向清晰：
  K1(High=10) → K2(High=12) → K3(High=11)
  High: 10 → 12 → 11  ← 先升后降，K2是高点   ✅ 清晰的顶分型

有包含关系时，方向模糊：
  K1(H=10,L=6) → K2(H=14,L=5)  ← K2 包含 K1
  高点升高了（10→14），低点也降低了（6→5）
  → 到底是涨还是跌？无法判断   ❌ 信息模糊
```

去除包含关系后，每根 K 线的高低点关系变得**单调**——要么"都在变高"，要么"都在变低"。这是后续分型识别的基础。

#### 包含关系的处理规则

处理包含关系需要先确定**方向**，再根据方向合并：

```
Step 1: 确定方向（用 K1 和 K2 的关系判断）

  K1.High < K2.High → 向上
  K1.High > K2.High → 向下
  K1.High = K2.High → 不处理（保持原样）

Step 2: 根据方向合并 K2 和 K3

  向上合并（取"高高"）：                向下合并（取"低低"）：
  ┌─┐                                  ┌─┐
  │ │ High = max(K2.H, K3.H)          │ │
  │ │                                  │ │ High = min(K2.H, K3.H)
  │ │ Low  = max(K2.L, K3.L)          │ │
  │ │                                  │ │ Low  = min(K2.L, K3.L)
  └─┘                                  └─┘

  记忆口诀：
  "向上取高高"  → 高点取大的，低点也取大的
  "向下取低低"  → 高点取小的，低点也取小的
```

#### 实际处理示例

```
原始K线：         处理后：

   K3
   ┌┐              合并后
K2 ││ K4           ┌┐
┌┐ ││ ┌┐         K2'││ K4
││ ││ ││   →      ││ ││ ┌┐
││ ││ ││          ││ ││ ││
└┘ └┘ └┘          └┘ └┘ └┘
K1                K1

K2和K3有包含关系（K3包含K2），方向向上
→ 合并为K2'：High=max(K2.H,K3.H), Low=max(K2.L,K3.L)
```

处理后的 K 线在 CZSC 中用 `NewBar` 对象表示。`NewBar` 与 `RawBar` 结构相似，但多了一个 `elements` 字段，记录合并前的原始 K 线。

### 3.3 分型（Level 3）

#### 分型的定义

**分型（FX，Fractal）**是三根**无包含** K 线组成的特定形态，是缠论分析的最小结构单元。

分型分为两种——**顶分型**和**底分型**：

```
顶分型（Mark.G）：中间K线的高点和低点都是三根中最高的

        K2
       ╱  ╲
   K1 ╱    ╲ K3             条件：K1.High < K2.High > K3.High
     ╱      ╲                     K1.Low  < K2.Low  > K3.Low
                                  
    → 分型值 fx = K2.High（顶分型取最高点）


底分型（Mark.D）：中间K线的高点和低点都是三根中最低的

   K1        K3
     ╲      ╱               条件：K1.High > K2.High < K3.High
      ╲    ╱                       K1.Low  > K2.Low  < K3.Low
       ╲  ╱
        K2                   → 分型值 fx = K2.Low（底分型取最低点）
```

> ⚠️ **常见误区**：分型要求中间 K 线的 **High 和 Low 都**满足条件，缺一不可。仅仅 High 满足而 Low 不满足，不构成分型。

#### 分型的力度

分型有强弱之分，力度越强，其标志的转折越可靠：

```
以底分型为例（K1-K2-K3，K2为底）：

强分型 ⭐⭐⭐：K3.Close > K1.High
  → 第三根K线收盘价超过第一根K线最高价，反转力度最强

  K1        K3
    ╲     ╱  ← K3.Close 在这里（超过K1.High）
     ╲   ╱
      ╲ ╱
       K2

中分型 ⭐⭐：K3.Close > K2.High（但 < K1.High）
  → 第三根K线收盘价超过底部K线最高价

弱分型 ⭐：K3.Close ≤ K2.High
  → 第三根K线收盘价甚至没有超过底部K线最高价，反转信号弱
```

#### 分型的实际意义

- **顶分型**暗示价格可能从上涨转向下跌（卖出信号）
- **底分型**暗示价格可能从下跌转向上涨（买入信号）
- 分型**不是**买卖点本身，而是识别**笔**的起点和终点

### 3.4 笔（Level 4）

#### 笔的定义

**笔（BI，Stroke）**是连接相邻顶分型和底分型的线段，是缠论中最重要的分析单元。

```
向上笔：从底分型连接到顶分型        向下笔：从顶分型连接到底分型

         G（顶分型）                  G（顶分型）
        ╱                              ╲
       ╱                                ╲
      ╱                                  ╲
     ╱                                    ╲
    D（底分型）                            D（底分型）

  方向：Direction.Up                方向：Direction.Down
  起点：fx_a = D（底分型）          起点：fx_a = G（顶分型）
  终点：fx_b = G（顶分型）          终点：fx_b = D（底分型）
```

#### 笔的识别条件

一笔成立需要同时满足两个条件：

| 条件 | 说明 | 原因 |
|------|------|------|
| **长度条件** | 起始分型到结束分型之间至少 6 根无包含 K 线 | 防止噪音产生过多细碎的"笔" |
| **价格条件** | 起始分型与结束分型的价格区间无包含关系 | 确保笔确实代表一段有意义的价格运动 |

```
有效笔 ✅                    无效笔 ❌（K线数不足）

  G                            G
  │╲                           ╲
  │ ╲                           ╲
  │  ╲       ≥ 6根K线            ╲   只有3根K线
  │   ╲                          ╲
  │    ╲                          D
  │     D
```

> 💡 **为什么要求最少 6 根 K 线？** 这是缠论对"最小有意义走势"的定义。太短的波动是噪音而非走势，不值得分析。你可以通过环境变量 `czsc_min_bi_len` 调整这个阈值（默认 6，老笔标准为 7）。

#### 笔的属性

每根笔携带丰富的分析信息：

| 属性 | 含义 | 用途 |
|------|------|------|
| `direction` | 方向（Up/Down） | 判断当前走势方向 |
| `high` / `low` | 笔的最高/最低价 | 画图、中枢计算 |
| `power_price` | 价格力度（起终点价差） | 笔的"幅度"度量 |
| `power_volume` | 成交量力度 | 笔的"能量"度量 |
| `length` | 笔包含的无包含 K 线数 | 笔的"时间"度量 |
| `change` | 涨跌幅 | 标准化的力度衡量 |
| `slope` | 斜率 | 笔的"速度"度量 |

### 3.5 中枢（Level 5）

#### 中枢的定义

**中枢（ZS，Hub/Pivot Zone）**是由至少三笔构成的价格震荡区间，代表多空力量的**均衡区域**。

```
中枢构成（至少3笔）：

价格
  │        G1          G2
  │       ╱  ╲       ╱  ╲
  ├──────╱─ZG─╲─────╱────╲──── 中枢上沿（ZG）
  │     ╱      ╲   ╱      ╲
  │    ╱  中枢区域╲╱        ╲
  ├──╱─────ZD────────────────── 中枢下沿（ZD）
  │ ╱              
  │D0          D1          
  │
  └──────────────────────────→ 时间

  ZG = min(G1.high, G2.high)   ← 前三笔高点的最小值
  ZD = max(D0.low, D1.low)     ← 前三笔低点的最大值
  ZZ = (ZG + ZD) / 2           ← 中枢中轴

  有效条件：ZG >= ZD（重叠区域存在）
```

> 💡 **类比理解**：中枢就像一个"价格磁场"——当价格在这个区间内来回震荡时，说明多空双方力量旗鼓相当，谁也无法取得压倒性优势。一旦价格突破中枢边界，意味着一方力量占优，可能产生趋势性机会。

#### 中枢与走势分类

有了中枢，就能对走势做客观分类：

```
上涨趋势：价格依次突破多个升高的中枢

价格
  │                    ┌──ZS3──┐
  │              ┌─ZS2─┤       │
  │        ┌ZS1──┤     │       │
  │     ╱  │     │     └───────┘
  │    ╱   │     └─────┘
  │   ╱    └─────┘
  │──╱──────────────────────────→ 时间

下跌趋势：价格依次跌破多个降低的中枢
盘整：只有一个中枢，价格在中枢内部反复震荡
```

把五个概念串联起来，缠论的分析流程如下：

```
原始K线 (RawBar)
    │
    ▼

└─────────────────────┘
    │

    │
    ▼
┌─────────────────────┐

└─────────────────────┘
    │
    ▼
┌─────────────────────┐
│   识别中枢 (ZS)      │  "发现均衡区"
│   → 走势分类         │
└─────────────────────┘
    │
    ▼
  买卖点判断
```

## 五、动手实践：你的第一次缠论分析

> 📝 以下代码可以直接运行，前提是已安装 CZSC 库

### 5.1 安装与环境准备

```bash
# 使用 pip 安装
pip install czsc

# 或使用 uv（推荐）
uv pip install czsc
```

### 5.2 第一个分析程序

```python
from czsc.core import CZSC, format_standard_kline, Freq
from czsc.mock import generate_symbol_kines

# Step 1: 生成模拟K线数据（30分钟周期，一个月）
df = generate_symbol_kines('000001', '30分钟', '20240101', '20240131')
print(f"生成了 {len(df)} 根原始K线")

# Step 2: 转换为 RawBar 对象列表
bars = format_standard_kline(df, freq=Freq.F30)

# Step 3: 创建 CZSC 分析对象（自动完成全部分析）
czsc_obj = CZSC(bars)

# Step 4: 查看分析结果
print(f"\n=== 分析结果 ===")
print(f"原始K线数量:    {len(czsc_obj.bars_raw)}")
print(f"无包含K线数量:  {len(czsc_obj.known)}")
print(f"分型数量:       {len(czsc_obj.fx_list)}")
print(f"笔数量:         {len(czsc_obj.bi_list)}")
```

### 5.3 逐步观察分析过程

```python
# 查看包含关系处理的效果
print("\n=== 包含关系处理 ===")
raw_count = len(czsc_obj.bars_raw)
merged_count = len(czsc_obj.known)
removed = raw_count - merged_count
print(f"原始 {raw_count} 根 → 处理后 {merged_count} 根（合并了 {removed} 根）")
print(f"包含比例: {removed / raw_count * 100:.1f}%")

# 查看分型
print("\n=== 分型列表（前5个） ===")
for fx in czsc_obj.fx_list[:5]:
    mark_name = "顶分型⬆" if fx.mark.value == "顶分型" else "底分型⬇"
    print(f"  {fx.dt} | {mark_name} | 价格={fx.fx:.2f} | 力度={fx.power_str}")

# 查看笔
print("\n=== 笔列表（前5根） ===")
for bi in czsc_obj.bi_list[:5]:
    dir_name = "向上↑" if bi.direction.value == "向上" else "向下↓"
    print(f"  {bi.fx_a.dt} → {bi.fx_b.dt}")
    print(f"    方向={dir_name} | 高={bi.high:.2f} 低={bi.low:.2f} | 幅度={bi.change:.2f}%")
```

### 5.4 运行结果解读

运行上述代码后，你会看到类似输出：

```
=== 分析结果 ===
原始K线数量:    248
无包含K线数量:  210
分型数量:       38
笔数量:         12

=== 包含关系处理 ===
原始 248 根 → 处理后 210 根（合并了 38 根）
包含比例: 15.3%

=== 分型列表（前5个） ===
  2024-01-02 10:30:00 | 底分型⬇ | 价格=9.85 | 力度=强
  2024-01-03 14:00:00 | 顶分型⬆ | 价格=10.82 | 力度=中
  ...
```

**解读要点**：
1. 约 15% 的 K 线存在包含关系——这在正常市场中是常见的
2. 分型总是顶底交替出现——这是算法保证的
3. 笔数量远少于分型数量——因为不是所有分型都能成笔

## 六、自测练习

### 练习 1（理解型 ⭐）：概念辨析

判断以下说法是否正确：

1. "缠论可以预测明天的股价" → ___
2. "顶分型一定是卖点" → ___
3. "包含关系处理后，K 线数量会减少或不变" → ___
4. "一笔至少需要 6 根无包含 K 线" → ___
5. "中枢的上沿 ZG 一定大于等于下沿 ZD" → ___

<details>
<summary>查看答案</summary>

1. ❌ 错误。缠论**描述**走势结构，不**预测**价格。
2. ❌ 错误。顶分型只是一个转折信号，它是识别**笔**的元素，不直接等于卖点。
3. ✅ 正确。包含关系处理只会合并 K 线，不会增加。
4. ✅ 正确。这是默认设置（`czsc_min_bi_len=6`）。
5. ✅ 正确。如果 ZG < ZD，说明不存在重叠区域，不构成有效中枢。

</details>

### 练习 2（应用型 ⭐⭐）：手动标注

在纸上画出以下 K 线序列（用高低点表示），手动标注包含关系和分型：

```
K线编号:  1    2    3    4    5    6    7
High:    10   12   11   14   12   11   13
Low:      8    9   10   11    9   10   11
```

提示：先处理包含关系，再找分型。

<details>
<summary>查看答案</summary>

1. **检查包含关系**：
   - K2 和 K3：K2(12,9) vs K3(11,10) → K2.H>K3.H 且 K2.L<K3.L → K2 包含 K3
   - 方向由 K1→K2 确定：K1.H(10) < K2.H(12) → 向上
   - 向上合并：High=max(12,11)=12, Low=max(9,10)=10 → 合并为 K2'(12,10)

2. **处理后序列**：K1(10,8), K2'(12,10), K4(14,11), K5(12,9), K6(11,10), K7(13,11)

3. **再次检查包含关系**（K5 和 K6）：
   - K5(12,9) vs K6(11,10)：K5.H>K6.H 且 K5.L<K6.L → K5 包含 K6
   - 方向由 K4→K5：K4.H(14)>K5.H(12) → 向下
   - 向下合并：High=min(12,11)=11, Low=min(9,10)=9 → K5'(11,9)

4. **最终序列**：K1(10,8), K2'(12,10), K4(14,11), K5'(11,9), K7(13,11)

5. **分型**：
   - K1, K2', K4：K1.H<K2'.H<K4.H → 无分型（K2' 不是极值）
   - K2', K4, K5'：K2'.H<K4.H>K5'.H 且 K2'.L<K4.L>K5'.L → **顶分型** ✅
   - K4, K5', K7：K4.H>K5'.H<K7.H 且 K4.L>K5'.L<K7.L → **底分型** ✅

</details>

### 练习 3（代码型 ⭐⭐）：修改参数观察变化

修改以下代码中的参数，观察对分析结果的影响：

```python
import os
from czsc.core import CZSC, format_standard_kline, Freq
from czsc.mock import generate_symbol_kines

# 实验1: 改变 K 线周期
for freq_name in ['5分钟', '30分钟', '日线']:
    df = generate_symbol_kines('000001', freq_name, '20240101', '20240331')
    freq_map = {'5分钟': Freq.F5, '30分钟': Freq.F30, '日线': Freq.D}
    bars = format_standard_kline(df, freq=freq_map[freq_name])
    c = CZSC(bars)
    print(f"{freq_name}: K线={len(c.bars_raw)}, 分型={len(c.fx_list)}, 笔={len(c.bi_list)}")

# 实验2: 改变最小笔长度
for min_len in [4, 6, 8, 10]:
    os.environ['czsc_min_bi_len'] = str(min_len)
    df = generate_symbol_kines('000001', '30分钟', '20240101', '20240331')
    bars = format_standard_kline(df, freq=Freq.F30)
    c = CZSC(bars)
    print(f"min_bi_len={min_len}: 笔数量={len(c.bi_list)}")
```

**观察**：
- 更大的 K 线周期 → 更少的分型和笔（"级别"更大）
- 更大的 `min_bi_len` → 更少的笔（过滤掉更多短笔）

## 七、常见误区

| # | 误区 | 正确理解 |
|---|------|----------|
| 1 | "分型就是最高点/最低点" | 分型是 3 根**无包含** K 线的**结构**，不是简单求极值 |
| 2 | "看到顶分型就卖" | 分型只是组成笔的元素，买卖点要看中枢和背驰 |
| 3 | "去掉包含关系会丢失信息" | 包含关系处理是合并而非删除，`elements` 字段保留了原始 K 线 |
| 4 | "缠论只能用于 A 股" | 缠论适用于任何有 K 线数据的市场：股票、期货、外汇、数字货币 |
| 5 | "笔越短越精确" | 过短的笔是噪音，适当的最小长度限制反而提高分析质量 |
