## 一、包含关系处理：算法精解

### 1.1 为什么这一步至关重要？

包含关系处理是缠论的**地基**。如果地基打歪了，上面所有的分析都会出错。

> **问题背景**：原始 K 线中经常出现一根大阳线"包住"前面的小阴线，或者一根大阴线"包住"后面的小阳线。这种包含关系会产生"虚假分型"，干扰后续分析。

> **设计决策**：缠论选择在分析之前，先用确定性规则合并包含 K 线，得到一个"清晰"的 K 线序列。这确保了后续每一步分析的输入都是"无歧义"的。

### 1.2 完整算法——三步法

包含关系处理使用 `remove_include(k1, k2, k3)` 函数，输入三根 K 线，输出是否存在包含以及处理后的 K 线。

> 💡 **设计决策**：为什么是三根 K 线而不是两根？因为判断包含关系的合并方向需要 k1 和 k2 的关系，而合并操作发生在 k2 和 k3 之间。k1 提供"上下文"，k2-k3 是"操作对象"。

#### Step 1：确定方向

```
用 k1 和 k2 的高点关系确定当前方向：

  k1.high < k2.high  →  方向向上 (Direction.Up)
  k1.high > k2.high  →  方向向下 (Direction.Down)
  k1.high = k2.high  →  不处理，k3 保持原样

          k2                           k1
         ╱  ╲                         ╱  ╲
    k1  ╱    ╲                  k2  ╱    ╲
       ╱      ╲                   ╱      ╲
   方向向上 ↑                  方向向下 ↓
```

> 💡 **为什么用高点判断方向？** 高点是多方力量的表现。k2 的高点高于 k1，说明多方在推动价格上行，趋势向上；反之说明空方占优。

#### Step 2：判断 k2 和 k3 是否存在包含关系

```
包含关系条件（任一成立即为包含）：

  条件A: k2.high >= k3.high AND k2.low <= k3.low   （k2 包含 k3）
  条件B: k2.high <= k3.high AND k2.low >= k3.low   （k3 包含 k2）

不存在包含关系 → 返回 (False, k3)，k3 作为新的独立 K 线
存在包含关系   → 进入 Step 3
```

#### Step 3：根据方向合并

```python
"""
合并规则的精确定义：

  向上合并（"高高原则"）：
    new.high = max(k2.high, k3.high)    # 取两者高的高点
    new.low  = max(k2.low, k3.low)      # 取两者高的低点
    new.dt   = k2.dt if k2.high > k3.high else k3.dt  # 时间跟高点走

  向下合并（"低低原则"）：
    new.high = min(k2.high, k3.high)    # 取两者低的高点
    new.low  = min(k2.low, k3.low)      # 取两者低的低点
    new.dt   = k2.dt if k2.low < k3.low else k3.dt    # 时间跟低点走

  公共处理：
    new.vol    = k2.vol + k3.vol        # 成交量求和
    new.amount = k2.amount + k3.amount  # 成交额求和
    new.elements = k2.elements + [k3]   # 记录原始 K 线
"""
```

#### 为什么向上取高高、向下取低低？

> **直觉理解**：在上升趋势中，我们关注多方的力量——多方能推到的最高点和多方"守住"的最高低点，都说明了多方的实力。在下降趋势中，我们关注空方的力量——空方打到的最低点和空方"压制住"的最低高点。

#### 完整示例

```
原始 K 线序列（标注 High, Low）：

  K1(10,6)  K2(14,9)  K3(12,10)  K4(8,5)  K5(11,7)

=== 处理过程 ===

1) K1 → 直接加入（无包含 K 线列表仅有 K1）
   结果：[K1(10,6)]

2) K2 → 列表不足2根，直接加入
   结果：[K1(10,6), K2(14,9)]

3) K3 → 取 k1=K1(10,6), k2=K2(14,9)
   方向：K1.high(10) < K2.high(14) → 向上
   包含检查：K2(14,9) vs K3(12,10)
     K2.high(14) >= K3.high(12) ✅ 且 K2.low(9) <= K3.low(10) ✅ → 有包含
   向上合并：High=max(14,12)=14, Low=max(9,10)=10 → K2'(14,10)
   结果：[K1(10,6), K2'(14,10)]

4) K4 → 取 k1=K1(10,6), k2=K2'(14,10)
   方向：K1.high(10) < K2'.high(14) → 向上
   包含检查：K2'(14,10) vs K4(8,5) → 无包含（14>8 但 10>5）
   结果：[K1(10,6), K2'(14,10), K4(8,5)]

5) K5 → 取 k1=K2'(14,10), k2=K4(8,5)
   方向：K2'.high(14) > K4.high(8) → 向下
   包含检查：K4(8,5) vs K5(11,7) → K5 包含 K4
   向下合并：High=min(8,11)=8, Low=min(5,7)=5 → K4'(8,5)
   结果：[K1(10,6), K2'(14,10), K4'(8,5)]

=== 最终序列 ===
  K1(10,6) → K2'(14,10) → K4'(8,5)
  
  这三根 K 线构成一个明确的顶分型（K2' 高低都高于两侧）
```

### 1.3 特殊情况处理

| 场景 | 处理方式 | 原因 |
|------|----------|------|
| k1.high == k2.high | 不处理包含，k3 原样加入 | 无法确定方向 |
| 连续多根包含 | 依次合并（每次用最新的合并结果） | `elements` 字段记录所有被合并的 K 线 |
| `elements` 超过 100 根 | 截断为最近 100 根 | 防止极端行情下内存溢出 |

### 1.4 代码实现印证

```python
from czsc.py.analyze import remove_include
from czsc.core import NewBar, Direction
from datetime import datetime

# 创建三根 K 线
k1 = NewBar(symbol="test", id=1, dt=datetime(2024,1,1), freq="30分钟",
            open=6, close=10, high=10, low=6, vol=100, amount=800, elements=[])
k2 = NewBar(symbol="test", id=2, dt=datetime(2024,1,2), freq="30分钟",
            open=9, close=14, high=14, low=9, vol=150, amount=1200, elements=[])
k3_raw = RawBar(symbol="test", id=3, dt=datetime(2024,1,3), freq="30分钟",
                open=10, close=12, high=12, low=10, vol=120, amount=960)

# 调用包含关系处理
has_include, result = remove_include(k1, k2, k3_raw)
print(f"有包含关系: {has_include}")        # True
print(f"合并后 High: {result.high}")       # 14
print(f"合并后 Low: {result.low}")         # 10
print(f"合并后 Vol: {result.vol}")         # 270 (150+120)
```

## 二、分型识别：精确定义与力度分析

### 2.1 分型的严格数学定义

分型由三根**连续的无包含 K 线**组成，根据中间 K 线与两侧 K 线的关系，分为顶分型和底分型。

```python
"""
顶分型 (Mark.G) 的完整条件：
  ① k1.high < k2.high    (K2 的高点比左侧高)
  ② k2.high > k3.high    (K2 的高点比右侧高)
  ③ k1.low  < k2.low     (K2 的低点比左侧高)
  ④ k2.low  > k3.low     (K2 的低点比右侧高)
  
  四个条件缺一不可！

底分型 (Mark.D) 的完整条件：
  ① k1.high > k2.high    (K2 的高点比左侧低)
  ② k2.high < k3.high    (K2 的高点比右侧低)
  ③ k1.low  > k2.low     (K2 的低点比左侧低)
  ④ k2.low  < k3.low     (K2 的低点比右侧低)
"""
```

> ⚠️ **关键细节**：很多教程只强调高点条件（①②），忽略低点条件（③④）。在 CZSC 的严格实现中，四个条件必须**同时成立**。这排除了一些"伪分型"——比如 K2 高点最高但低点不是最高的情况。

### 2.2 分型力度判断

分型力度反映了转折的"可信度"。CZSC 从两个维度评估：

#### 价格力度（power_str）

以**底分型**为例（K1-K2-K3，K2 为底）：

```
强分型 ⭐⭐⭐                中分型 ⭐⭐                  弱分型 ⭐

K1       K3                K1       K3                K1
 ╲     ┌╱ K3.Close        ╲     ╱                    ╲       K3
  ╲   ╱│  > K1.High        ╲   ╱ K3.Close            ╲     ╱ K3.Close
   ╲ ╱ │                    ╲ ╱  > K2.High             ╲   ╱  ≤ K2.High
    K2  │                    K2                          ╲ ╱
        │                                                K2
```

以**顶分型**为例（K1-K2-K3，K2 为顶），规则对称：
- **强**：K3.Close < K1.Low
- **中**：K3.Close < K2.Low
- **弱**：K3.Close ≥ K2.Low

#### 成交量力度（power_volume）

```python
fx.power_volume = sum(k.vol for k in fx.elements)  # 三根 K 线成交量之和
```

成交量力度没有绝对标准，通常用于**同级别分型对比**——同类分型中，成交量力度越大的越可靠。

#### 分型内部结构（has_zs）

```python
fx.has_zs = min(k1.high, k2.high, k3.high) >= max(k1.low, k2.low, k3.low)
```

如果三根 K 线存在重叠区间（`has_zs=True`），说明分型内部有一个"微型中枢"，转折的力度更强。

### 2.3 分型序列的约束

CZSC 在识别分型时，**强制要求顶底交替**：

```
有效序列：  D → G → D → G → D → G
无效序列：  D → D → G  ← 连续两个底分型，丢弃第二个

"当连续出现两个相同标记的分型时，保留第一个，丢弃第二个"
```

> **为什么？** 顶底交替是笔的组成前提。如果允许连续顶分型，就无法确定笔的起终点。

## 三、笔的识别：从分型到走势

### 3.1 笔的完整识别算法

笔的识别是缠论中最复杂的部分，下面逐步拆解。

#### Step 1：确定起始分型

```
从分型序列中取第一个分型 fx_a：
  - 如果 fx_a 是底分型 → 要找的笔方向为向上 → 寻找后面的顶分型
  - 如果 fx_a 是顶分型 → 要找的笔方向为向下 → 寻找后面的底分型
```

#### Step 2：寻找结束分型

```python
"""
寻找结束分型 fx_b 的规则：

向上笔（fx_a 为底分型 D）：
  在 fx_a 之后的所有顶分型中，找满足以下条件的：
  ① fx_b.dt > fx_a.dt          （时间在 fx_a 之后）
  ② fx_b.fx > fx_a.fx          （价格高于起始分型）
  ③ 当有多个候选时，取 fx_b.high 最大的  （取最高的顶分型）

向下笔（fx_a 为顶分型 G）：
  在 fx_a 之后的所有底分型中，找满足以下条件的：
  ① fx_b.dt > fx_a.dt          （时间在 fx_a 之后）
  ② fx_b.fx < fx_a.fx          （价格低于起始分型）
  ③ 当有多个候选时，取 fx_b.low 最小的   （取最低的底分型）
"""
```

> 💡 **为什么取最高/最低？** 这是"贪心"策略——在所有可能的笔终点中，选择幅度最大的。这避免了短笔被更长的走势"淹没"的问题。

#### Step 3：验证成笔条件

```
条件 1：长度验证
  笔（从 fx_a 第一根元素到 fx_b 第三根元素）的无包含 K 线数量 ≥ min_bi_len
  默认 min_bi_len = 6

条件 2：价格区间无包含
  不能出现：fx_a 的高低范围完全包含 fx_b 的高低范围（或反过来）
  
  具体判断：
  ab_include = (a.high > b.high AND a.low < b.low) OR
               (a.high < b.high AND a.low > b.low)
  
  如果 ab_include 为 True → 这不是一笔有效的笔

两个条件都满足 → 成笔 ✅
任一条件不满足 → 不成笔 ❌，继续寻找
```

### 3.2 笔的属性深度解析

#### 笔的力度度量

CZSC 为每根笔计算了多种力度指标：

```python
class BI:
    @property
    def power_price(self):
        """价格力度：笔的高低点之差"""
        return self.high - self.low
    
    @property
    def power_volume(self):
        """成交量力度：笔内所有 K 线成交量之和"""
        return sum(bar.vol for bar in self.bars)
    
    @property
    def change(self):
        """涨跌幅：笔起终点的价格变化百分比"""
        return (self.fx_b.fx - self.fx_a.fx) / self.fx_a.fx * 100
    
    @property
    def length(self):
        """时间跨度：笔包含的无包含 K 线数量"""
        return len(self.bars)
    
    @property
    def slope(self):
        """斜率：价格变化除以时间跨度"""
        return self.power_price / self.length if self.length else 0
    
    @property
    def rsq(self):
        """R² 拟合优度：衡量笔内部走势的线性程度（0~1）"""
        # 越接近 1，走势越"直"
        # 越接近 0，走势越"曲折"
```

#### 笔的内部结构（fake_bis）

每根笔内部可能包含更小级别的波动，CZSC 用 `fake_bis` 属性描述这些"次级别笔"：

```
一根向上笔的内部结构：

         G (终点)
        ╱
       ╱  ← fake_bi_3 (向上)
      ╱
     x    ← 小级别回调
    ╱
   ╱      ← fake_bi_1 (向上)
  ╱
 D (起点)

fake_bis 就是笔内部的小级别波动
它们不是正式的"笔"（因为不满足最小长度要求），但可以辅助判断笔的质量
```

### 3.3 笔的更新与破坏

这是整个缠论算法中**最精妙**也**最容易出错**的部分。理解它需要建立一个关键的心智模型：

> 💡 **心智模型——"铅笔画线"**：想象你在纸上用铅笔画一根线。你从底部开始向上画，画到某个点时你觉得"可能到顶了"，就停下标记一个顶分型。但如果下一刻价格又创了新高，你就需要用橡皮擦掉之前的标记，继续向上画。这个"擦掉重画"的过程就是**笔破坏**。

#### 笔的破坏（Bi Destruction）

```
什么是笔破坏？

假设最后一笔是向上笔（D→G），之后又来了新的 K 线：

情况1：新 K 线的高点超过了 G 的高点
  → 这根向上笔还没结束！之前判断的 G 是"假顶"
  → 需要：回退笔，重新寻找真正的顶

情况2：新 K 线的低点低于 D 的低点（跌破了笔起点）
  → 之前判断的向上笔完全无效
  → 需要：删除这根笔，回退到更早的状态重新分析

处理流程：
  1. 发现破坏：bars_ubi 的极值超过了 last_bi 的极值
  2. 回退：将 last_bi 的 K 线回退到 bars_ubi 中
  3. 删除：从 bi_list 中移除 last_bi
  4. 重新识别：在回退后的 bars_ubi 上重新寻找笔
```

```python
"""
笔破坏的精确条件：

last_bi = bi_list[-1]       # 最后一笔
bars_ubi = [...]             # 未完成笔的 K 线

if last_bi.direction == Direction.Up:
    # 向上笔被破坏：未完成 K 线出现了更高的高点
    if bars_ubi[-1].high > last_bi.high:
        destroy_and_rebuild()

if last_bi.direction == Direction.Down:
    # 向下笔被破坏：未完成 K 线出现了更低的低点
    if bars_ubi[-1].low < last_bi.low:
        destroy_and_rebuild()
"""
```

> ⚠️ **这意味着**：CZSC 的分析结果不是"一次确定"的。随着新 K 线的到来，之前已确认的笔可能被推翻。这正是"走势终完美"哲学的体现——我们只有在走势真正完成后，才能100% 确认其结构。

```python
@property

    衡量笔的"纯净程度"：
    - SNR 高 → 笔内部走势流畅，方向明确
    - SNR 低 → 笔内部波动大，可能是虚假信号

```

## 四、中枢理论：多空力量的均衡场

### 4.1 中枢的精确构造

#### 前三笔决定中枢

中枢由至少三笔构成，其核心参数由**前三笔**决定：

```
中枢构造规则：

  给定笔序列 [bi_0, bi_1, bi_2, ...]（至少3笔）

  ZG = min(bi_0.high, bi_1.high, bi_2.high)     # 前三笔高点的最小值
  ZD = max(bi_0.low, bi_1.low, bi_2.low)         # 前三笔低点的最大值
  ZZ = ZD + (ZG - ZD) / 2                        # 中枢中轴

  有效性条件：ZG >= ZD（重叠区域必须存在）

  示意图：
  价格
    │   bi_0.high         bi_2.high
    │      ╲                ╱
    ├───────╲────ZG────────╱────── 中枢上沿
    │        ╲            ╱
    │         ╲    ZZ    ╱        中枢中轴
    │          ╲        ╱
    ├───────────╲─ZD───╱────────── 中枢下沿
    │            ╲    ╱
    │          bi_1.low
    └──────────────────────────→ 时间
```

#### 中枢的完整参数

| 参数 | 计算方式 | 含义 |
|------|----------|------|
| `ZG` | min(前三笔 high) | 中枢上沿——多空共同认可的"高价区" |
| `ZD` | max(前三笔 low) | 中枢下沿——多空共同认可的"低价区" |
| `ZZ` | (ZG+ZD)/2 | 中枢中轴——多空的"均衡价格" |
| `GG` | max(所有笔 high) | 中枢区间最高点 |
| `DD` | min(所有笔 low) | 中枢区间最低点 |

> 💡 **为什么用前三笔的 min(high) 和 max(low)？**
>
> 这是取三笔的**重叠区间**。三笔中最低的高点就是所有笔都能触及的最高价——即 ZG；三笔中最高的低点就是所有笔都要经过的最低价——即 ZD。[ZD, ZG] 就是三笔共同经历的价格区间。

### 4.2 中枢的延伸与升级

#### 中枢延伸

当第四笔、第五笔……仍然与中枢区间有交集时，中枢**延伸**：

```
中枢延伸条件：new_bi 与 [ZD, ZG] 有交集
  即 new_bi.low <= ZG 且 new_bi.high >= ZD

延伸时：
  ZG、ZD 不变（由前三笔决定，锁定不变）
  GG = max(GG, new_bi.high)    # 最高点可能更新
  DD = min(DD, new_bi.low)     # 最低点可能更新
  bis 列表追加 new_bi

不延伸（new_bi 完全在中枢之外）：
  → 当前中枢结束
  → 后续笔可能形成新的中枢
```

#### 中枢升级

当一个方向上出现两个或更多依次抬高（或降低）的中枢时，形成**趋势**：

```
上涨趋势：两个依次升高的中枢

价格
  │              ┌───ZS2───┐
  │              │         │
  │         ┌ZS1─┤         │
  │         │    │         └─────
  │    ╱    │    └─────┘
  │   ╱     └─────┘
  │──╱─────────────────────────→ 时间

条件：ZS2.ZD > ZS1.ZG（第二个中枢的下沿高于第一个中枢的上沿）
```

### 4.3 走势分类——缠论的核心应用

有了中枢，就能对当前走势做**穷举分类**：

| 走势类型 | 中枢数量 | 中枢关系 | 操作策略 |
|----------|----------|----------|----------|
| **上涨趋势** | ≥ 2 | 依次升高 | 持有多头，回调不破中枢加仓 |
| **下跌趋势** | ≥ 2 | 依次降低 | 回避或做空，反弹不破中枢减仓 |
| **盘整上涨** | 1 | — | 中枢上方做多，但注意假突破 |
| **盘整下跌** | 1 | — | 中枢下方回避，等待方向明确 |
| **盘整** | 1 | — | 中枢内部高抛低吸 |

> **关键洞察**：这五种分类是**穷举**的——任何走势在任何时刻，必然属于其中之一。这消除了传统技术分析中"看不清方向"的困境。

## 五、买卖点理论

### 5.1 三类买点

缠论定义了三种标准买点，每种买点的逻辑和风险特征不同：

#### 第一类买点（一买）

```
一买：下跌趋势结束时出现的买点

原理：在下跌趋势中（两个依次降低的中枢），当最后一段下跌
      的"力度"弱于前一段（即发生背驰），说明空方力量衰竭，
      下跌趋势即将结束。

价格
  │╲
  │ ╲  ← 力量强
  │  ╲
  │   ╲  ┌ZS1┐
  │    ╲─┤   ├─╱╲
  │      └───┘    ╲  ← 力量弱（背驰）
  │                ╲
  │          ┌ZS2┐  ╲
  │          │   │   ★ ← 一买！
  │          └───┘
  │
  
特征：
  - 风险最大（在下跌中买入）
  - 盈利空间最大（如果判断正确）
  - 判断依据：力度背驰
```

#### 第二类买点（二买）

```
二买：一买后第一次回调的低点

原理：一买之后，价格通常会先反弹一段再回调。
      如果回调的低点不跌破一买的位置，说明趋势已经反转。

价格
  │
  │          ╱╲   ← 一买后的反弹
  │         ╱  ╲
  │    ★   ╱    ╲
  │    ↑  ╱      ★ ← 二买！（不跌破一买低点）
  │  一买╱       ↑
  │              回调低点
  
特征：
  - 风险中等（有一买作为参照）
  - 盈利空间中等
  - 判断依据：回调不破一买低点
```

#### 第三类买点（三买）

```
三买：价格突破中枢上沿后的回踩不跌回中枢

原理：价格突破中枢高点 ZG 后回踩，如果回踩低点
      不跌破 ZG，说明原中枢已成为支撑，上涨趋势确认。

价格
  │                ╱╲
  │               ╱  ╲
  │              ╱    ★ ← 三买！（回踩不破 ZG）
  │         ╱╲  ╱     ↑
  ├────ZG──╱──╲╱──────┤
  │       ╱   中枢     │
  ├──ZD──╱────────────┤
  │     ╱
  │
  
特征：
  - 风险最小（趋势已确认）
  - 盈利空间相对较小（介入点偏高）
  - 判断依据：回踩不破中枢上沿
```

### 5.2 三类卖点

卖点与买点完全对称：

| 卖点 | 条件 | 风险 |
|------|------|------|
| **一卖** | 上涨趋势背驰 | 最大（可能还涨） |
| **二卖** | 一卖后反弹高点不超一卖 | 中等 |
| **三卖** | 跌破中枢下沿后反弹不回中枢 | 最小（趋势已转） |

### 5.3 背驰判断

**背驰（Divergence）**是判断一买/一卖的核心依据。它的本质是"新的同向走势的力度弱于前一段"。

#### 用 MACD 辅助判断背驰

```
MACD 背驰判断（以底背驰为例）：

价格创新低，但 MACD 不创新低 → 底背驰 → 可能出现一买

价格 │     ╲              ╲
     │      ╲              ╲
     │       ╲    ╱╲        ╲
     │        ╲  ╱  ╲        ★ 新低
     │    旧低 ╲╱    ╲      ╱
     │                ╲    ╱
     │                 ╲  ╱
     │                  ╲╱
     │
MACD │   
     │   ──── 
     │  │绿柱│     ──── 
     │  │    │    │绿柱│← 面积更小！
     │  │    │    │    │
     └──┴────┴────┴────┴──→

结论：价格新低但 MACD 绿柱面积缩小 → 空方力量减弱 → 底背驰
```

> ⚠️ **注意**：MACD 只是辅助工具，不是唯一判据。缠论原文中用的是"区间套"方法精确定位背驰点，但 MACD 在实践中更简便。

### 练习 1（应用型 ⭐⭐）：包含关系处理

手工处理以下 K 线序列的包含关系：

```
K1: High=10, Low=6
K2: High=13, Low=8
K3: High=11, Low=9    ← K2和K3有包含吗？
K4: High=15, Low=10
K5: High=14, Low=11   ← K4和K5有包含吗？
K6: High=12, Low=7
```

<details>
<summary>查看答案</summary>

**Step 1: K1-K2-K3**
- 方向：K1.H(10)<K2.H(13) → 向上
- K2(13,8) vs K3(11,9)：K2.H>K3.H 且 K2.L<K3.L → **K2 包含 K3**
- 向上合并：H=max(13,11)=13, L=max(8,9)=9 → K2'(13,9)

**Step 2: K1-K2'-K4**
- K2'(13,9) vs K4(15,10)：无包含（K4 更高更低都不满足）
- 序列：K1(10,6), K2'(13,9), K4(15,10)

**Step 3: K2'-K4-K5**
- 方向：K2'.H(13)<K4.H(15) → 向上
- K4(15,10) vs K5(14,11)：K4.H>K5.H 且 K4.L<K5.L → **K4 包含 K5**
- 向上合并：H=max(15,14)=15, L=max(10,11)=11 → K4'(15,11)

**Step 4: K2'-K4'-K6**
- K4'(15,11) vs K6(12,7)：无包含
- 最终序列：K1(10,6), K2'(13,9), K4'(15,11), K6(12,7)

**分型检测**：
- K1,K2',K4'：1.H<K2'.H<K4'.H → 不是分型（K2' 不是极值）
- K2',K4',K6：K2'.H<K4'.H>K6.H 且 K2'.L<K4'.L>K6.L → **顶分型** ✅

</details>

```
案例A：底分型 D(low=8) → 7根无包含K线 → 顶分型 G(high=15)
案例B：底分型 D(low=8, high=16) → 8根无包含K线 → 顶分型 G(high=14, low=6)
```

### 练习 3（创造型 ⭐⭐⭐⭐）：设计笔质量评分

基于 CZSC 的笔属性（`power_price`、`power_volume`、`length`、`slope`、`rsq`、`SNR`），设计一个 0-100 的笔质量评分函数。

要求：
1. 考虑至少 3 个维度
2. 每个维度有明确的评分标准
3. 解释你的评分逻辑

<details>
<summary>参考思路</summary>

```python
def bi_quality_score(bi):
    """笔质量评分（满分100）"""
    score = 0
    
    # 维度1: 长度分 (30分) — 笔越长越可靠
    if bi.length >= 12:
        score += 30
    elif bi.length >= 8:
        score += 20
    elif bi.length >= 6:
        score += 10
    
    # 维度2: R² 拟合优度 (40分) — 走势越线性越干净
    score += int(bi.rsq * 40)
    
    # 维度3: 分型力度 (30分) — 起终点分型越强越可靠
    fx_score = 0
    if bi.fx_a.power_str == "强":
        fx_score += 15
    elif bi.fx_a.power_str == "中":
        fx_score += 10
    if bi.fx_b.power_str == "强":
        fx_score += 15
    elif bi.fx_b.power_str == "中":
        fx_score += 10
    score += fx_score
    
    return min(score, 100)
```

**设计理由**：
- 长度反映走势的"充分性"，过短的笔可能是噪音
- R² 反映走势的"纯净度"，曲折的笔表示多空反复拉锯
- 分型力度反映起终点的"确认程度"，强分型的转折更可信

</details>

## 七、常见问题

### Q1：为什么有时候分析结果会"变"？

CZSC 使用增量更新机制。每来一根新 K 线，都可能触发"笔破坏"，导致之前已确认的笔被推翻。这不是 bug，而是算法的设计——只有走势真正结束后，结构才会稳定。

**实践建议**：不要依赖"最后一笔"做决策。建议至少等到"倒数第二笔"确认后再行动。

### Q2：min_bi_len 设为多少合适？

| 值 | 效果 | 适用场景 |
|----|------|----------|
| 4-5 | 笔更多、更细碎 | 日内高频交易 |
| 6（默认） | 平衡 | 绝大多数场景 |
| 7 | "老笔"标准 | 严谨的中长线分析 |
| 8-10 | 笔更少、更稳定 | 只关注大级别走势 |

### Q3：中枢的 ZG 和 GG 有什么区别？

- **ZG（中枢上沿）**：前三笔的重叠区间上沿，代表"均衡区"的上界
- **GG（最高点）**：中枢所有笔的最高点，可能远高于 ZG

在实际操作中：
- 用 ZG/ZD 判断**趋势**和**买卖点**
- 用 GG/DD 判断**支撑/阻力**和**止损位**
