## 一、缠论基础概念

### 1.1 什么是缠论？

**缠中说禅（缠论）** 是一套由缠师（缠中说禅博主）创立的技术分析理论体系。它通过对价格走势的严格分类和定义，提供了一套客观的市场分析方法。

#### 核心思想

缠论的核心思想可以概括为：

1. **走势终完美**：所有级别的走势最终都会完成
2. **级别递归**：大级别走势由小级别走势递归构成
3. **买卖点客观**：通过分型、笔、中枢的严格定义，客观识别买卖点

#### 为什么选择缠论？

| 特点 | 传统技术分析 | 缠论 |
|------|-------------|------|
| 主观性 | 强（依赖经验） | 弱（规则明确） |
| 买卖点 | 模糊 | 精确到点 |
| 级别处理 | 单一 | 多级别联立 |
| 趋势判断 | 事后诸葛亮 | 客观分类 |

### 1.2 核心对象关系图

```
原始K线序列
    ↓
┌─────────────────────┐
│   去除包含关系处理   │  → NewBar（处理后K线）
└─────────────────────┘
    ↓
┌─────────────────────┐
│     识别分型（FX）    │  → 顶分型（G）/ 底分型（D）
└─────────────────────┘
    ↓
┌─────────────────────┐
│     识别笔（BI）     │  → 向上笔 / 向下笔
└─────────────────────┘
    ↓
┌─────────────────────┐
│     识别中枢（ZS）    │  → 价格震荡区间
└─────────────────────┘
    ↓
   买卖点判断
```

## 二、核心数据对象

### 2.1 RawBar（原始K线）

`RawBar` 是最基础的数据结构，表示一根原始的K线。

```python
from dataclasses import dataclass
from datetime import datetime
from czsc.py.enum import Freq

@dataclass
class RawBar:
    """原始K线元素"""
    symbol: str      # 标的代码
    id: int          # K线唯一标识（升序）
    dt: datetime     # K线时间
    freq: Freq       # K线频率
    open: float      # 开盘价
    close: float     # 收盘价
    high: float      # 最高价
    low: float       # 最低价
    vol: float       # 成交量
    amount: float    # 成交额
    cache: dict      # 用户缓存（可存储技术指标等）
```

#### 实用属性

```python
# 假设有一根K线
bar = RawBar(symbol="000001", id=1, dt=datetime.now(), freq=Freq.F30,
             open=10.0, close=10.5, high=10.8, low=9.9, vol=10000, amount=105000)

# 计算K线形态特征
print(f"上影线: {bar.upper}")    # high - max(open, close)
print(f"下影线: {bar.lower}")    # min(open, close) - low
print(f"实体: {bar.solid}")      # abs(open - close)
```

### 2.2 NewBar（处理后的K线）

`NewBar` 是经过**包含关系处理**后的K线。

#### 什么是包含关系？

当两根K线之间存在"包含"关系时，需要合并处理：

- **高点包含**：一根K线的高点在另一根的高低点范围内
- **低点包含**：一根K线的低点在另一根的高低点范围内

#### 处理原则

```python
"""
包含关系处理原则（缠论定义）：

1. 确定方向：根据前两根无包含关系K线的高点关系确定趋势方向
   - K1.high < K2.high → 方向向上（Up）
   - K1.high > K2.high → 方向向下（Down）

2. 合并规则：
   - 向上时：取两根K线的较高高点、较高低点
   - 向下时：取两根K线的较低高点、较低低点
"""
```

### 2.3 FX（分型）

**分型**是缠论中的基础形态，是识别笔的起点。

#### 分型的定义

```python
@dataclass
class FX:
    """分型对象"""
    symbol: str      # 标的代码
    dt: datetime     # 分型时间（中间K线的时间）
    mark: Mark       # 分型标记：G=顶分型，D=底分型
    high: float      # 分型高点
    low: float       # 分型低点
    fx: float        # 分型点（顶分型为high，底分型为low）
    elements: List   # 构成分型的三根K线
    cache: dict      # 用户缓存
```

#### 顶分型与底分型

```python
"""
顶分型（Mark.G）：
    K1.high < K2.high > K3.high
    且 K1.low < K2.low > K3.low

底分型（Mark.D）：
    K1.low > K2.low < K3.low
    且 K1.high > K2.high < K3.high

        顶分型            底分型
           /\              /\
          /  \            /  \
    K1   /    \ K3   K3  /    \ K1
        /      \          /      \
       K2      K        K2      K
```

#### 分型力度

```python
# FX 对象提供了分型力度判断
fx.power_str  # 返回 "强"/"中"/"弱"
fx.power_volume  # 返回成交量力度
```

**力度判断标准**（以底分型为例）：
- **强**：第三根K线收盘价 > 第一根K线高点
- **中**：第三根K线收盘价 > 第二根K线高点
- **弱**：第三根K线收盘价 ≤ 第二根K线高点

### 2.4 BI（笔）

**笔**是连接相邻分型的线段，是缠论分析的核心单元。

```python
@dataclass
class BI:
    """笔对象"""
    symbol: str          # 标的代码
    fx_a: FX             # 起始分型
    fx_b: FX             # 结束分型
    direction: Direction # 笔的方向：UP/DOWN
    power_price: float   # 价格力度（绝对值）
    power_vol: float     # 成交量力度
    elements: List       # 构成笔的分型列表
    cache: dict          # 用户缓存
```

#### 笔的识别条件

```python
"""
笔的识别条件（严格）：

1. 起始分型到结束分型之间，至少跨越一根K线
2. 满足最小笔长度要求（由 envs.czsc_min_bi_len 控制）
3. 顶分型和底分型必须交替出现
4. 结束分型的极值必须超过起始分型的极值

向上笔：底分型 → 顶分型，且 fx_b.fx > fx_a.fx
向下笔：顶分型 → 底分型，且 fx_b.fx < fx_a.fx
"""
```

### 2.5 ZS（中枢）

**中枢**是价格震荡的密集区域，是判断趋势和买卖点的关键。

```python
@dataclass
class ZS:
    """中枢对象"""
    symbol: str          # 标的代码
    zs_dt: datetime      # 中枢形成时间
    start_dt: datetime   # 中枢开始时间
    end_dt: datetime     # 中枢结束时间
    bid: str             # 中枢唯一标识
    direction: Direction # 中枢方向：UP/DOWN
    lines: List[BI]      # 构成中枢的笔
    zz: float            # 中枢中轴（ZG + ZD）/ 2
    zg: float            # 中枢高点（中枢区间上限）
    zd: float            # 中枢低点（中枢区间下限）
```

#### 中枢的构成

```python
"""
中枢定义：

1. 至少由三笔构成：上-下-上 或 下-上-下
2. 三个笔的区间有重叠部分
3. 重叠部分形成中枢区间 [ZD, ZG]

     ZG（中枢高点）
        │
   ┌─────┴─────┐
   │   中枢    │
   │           │
   └─────┬─────┘
        │
     ZD（中枢低点）

向上中枢：下-上-下-上... 笔序列构成
向下中枢：上-下-上-下... 笔序列构成
"""
```

## 三、信号-事件-交易三层体系

### 3.1 体系架构

CZSC 采用**信号-事件-交易**三层量化体系，这是系统设计的核心。

```
┌─────────────────────────────────────────────────────────┐
│                      交易决策层                          │
│  (Position: 持仓策略、开平仓条件、风控、止盈止损)        │
└─────────────────────────────────────────────────────────┘
                           ↑
┌─────────────────────────────────────────────────────────┐
│                       事件组合层                         │
│  (Event: signals_all, signals_any, signals_not)         │
└─────────────────────────────────────────────────────────┘
                           ↑
┌─────────────────────────────────────────────────────────┐
│                       信号计算层                         │
│  (Signal: 均线方向、MACD、分型状态、成交量特征...)       │
└─────────────────────────────────────────────────────────┘
```

### 3.3 Event（事件）

**事件**是信号的逻辑组合，通过 AND/OR/NOT 运算描述更复杂的交易条件。

```python
@dataclass
class Event:
    """事件对象"""
    name: str                  # 事件名称
    operate: Operate           # 操作类型：开多/平多/开空/平空
    signals_all: List[Signal]  # 必须全部满足的信号
    signals_any: List[Signal]  # 满足任一即可的信号
    signals_not: List[Signal]  # 不能出现的信号
```

#### 事件逻辑示例

```python
"""
事件示例：三买信号

满足条件：

    "日线_cxt_third_buy_符合_0",      # 满足三买定义
    "日线_tas_macd_base_红柱_0",      # MACD红柱

signals_any = [
    "30分钟_cxt_first_buy_符合_0",    # 30分钟出现一买
```

```python
    hold_max: int                # 最大持仓时间
    t1: Optional[datetime]       # 开仓时间
    t2: Optional[datetime]       # 平仓时间
    operated: Operate            # 当前操作
```

## 四、CZSC 分析类

### 4.1 CZSC 类概述

`CZSC` 是缠论分析的核心类，负责K线数据的处理和分析。

```python
class CZSC:
    """缠中说禅分析器"""

    def __init__(self, bars: List[RawBar], max_bi_num: int = 50,
                 get_bi: Callable = None, get_zs: Callable = None):
        """
        初始化分析器

        参数：
            bars: 原始K线列表
            max_bi_num: 最大保留笔数量
            get_bi: 自定义笔识别函数
            get_zs: 自定义中枢识别函数
        """
```

### 4.2 核心属性

```python
# CZSC 对象的核心属性
czsc_obj.bars_raw    # List[RawBar]  原始K线列表
czsc_obj.bars_away   # List[RawBar]  未参与计算的K线（用于初始化）
czsc_obj.known       # List[NewBar]  去除包含关系后的K线
czsc_obj.fx_list     # List[FX]      分型列表
czsc_obj.bi_list     # List[BI]      笔列表
czsc_obj.zs_list     # List[ZS]      中枢列表
czsc_obj.symbol      # str           标的代码
czsc_obj.freq        # Freq          K线频率
```

### 4.3 使用示例

```python
from czsc.core import CZSC, format_standard_kline, Freq
from czsc.mock import generate_symbol_kines

# 生成测试数据
df = generate_symbol_kines('000001', '30分钟', '20240101', '20240131')
bars = format_standard_kline(df, freq=Freq.F30)

# 创建CZSC分析对象
czsc_obj = CZSC(bars)

# 查看分析结果
print(f"原始K线: {len(czsc_obj.bars_raw)} 根")
print(f"处理后K线: {len(czsc_obj.known)} 根")
print(f"分型数量: {len(czsc_obj.fx_list)} 个")
print(f"笔数量: {len(czsc_obj.bi_list)} 个")
print(f"中枢数量: {len(czsc_obj.zs_list)} 个")

# 查看最新状态
if czsc_obj.bi_list:
    last_bi = czsc_obj.bi_list[-1]
    print(f"\n最新笔:")
    print(f"  方向: {last_bi.direction}")
    print(f"  起点: {last_bi.fx_a.dt}, 价格: {last_bi.fx_a.fx}")
    print(f"  终点: {last_bi.fx_b.dt}, 价格: {last_bi.fx_b.fx}")
```

### 4.4 更新机制

CZSC 对象支持增量更新，适合实时分析场景。

```python

# 获取新K线后，可以使用 update 方法

czsc_obj.update(new_bar)

# 更新后，所有分析结果会自动更新
print(f"更新后笔数量: {len(czsc_obj.bi_list)}")
```

```python
czsc_30m = CZSC(bars_30m)
czsc_daily = CZSC(bars_daily)

# 多级别综合判断
if (czsc_daily.bi_list[-1].direction == Direction.Up and
    czsc_30m.bi_list[-1].direction == Direction.Up):
    print("日线和30分钟级别都是向上笔，考虑做多")
```

## 六、实战案例

### 6.1 完整分析流程

```python
from czsc.core import CZSC, BarGenerator, format_standard_kline, Freq

# 1. 准备数据

# 3. 配置信号
signals_config = [
    {'name': 'czsc.signals.tas_ma_base_V221101',

    {'name': 'czsc.signals.cxt_fx_power_V221107',
     'freq': '30分钟', 'di': 1},
]

# 4. 计算信号
cs = CzscSignals(bg, signals_config=signals_config)

# 5. 查看结果
print("当前信号:")
for key, value in cs.s.items():
    if '信号' in key:
        print(f"  {key}: {value}")
```

## 七、常见问题

### Q1: 为什么我的分型识别不对？

A: 检查以下几点：
1. 确认K线数据已经过包含关系处理
2. 检查K线是否按时间严格升序排列
3. 确认 `id` 字段是否唯一且递增
4. 查看 `envs.czsc_min_bi_len` 参数设置

### Q2: 如何调整笔的识别灵敏度？

A: 可以通过环境变量调整：
```python
from czsc import envs

# 设置最小笔长度（默认为 6）
envs.set_min_bi_len(3)

# 设置最大保留笔数量
envs.set_max_bi_num(100)
```

### Q3: 实时分析时如何更新CZSC对象？

A: 使用 `update` 方法：
```python
# 实时场景
czsc_obj = CZSC(initial_bars)

while True:
    new_bar = get_new_bar()  # 获取新K线
    czsc_obj.update(new_bar)

    # 处理最新状态
    process_signals(czsc_obj)
```
